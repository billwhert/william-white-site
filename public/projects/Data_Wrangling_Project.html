Real-world Data Wrangling

1. Gather data

In this section, we will extract data using two different data gathering methods and combine the data. We will use a hardcopy CSV file as well as an appropriate API to retrieve data programmatically.

1.1. Problem Statement

In this project, we will be determining if player engagement (e.g., playtime) for games available on the PC platform Steam is associated with lifetime, global sales. With this information, game developers and advertisers/marketers could determine games that are best for development, potential popularity for streaming and promotional partnerships, and factors related to creating games that could be considered generally "popular". I will analyze two datasets to answer this question: lifetime sales data for games and Steam engagement metrics.

1.2. Gather at least two datasets using two different data gathering methods

Dataset 1: Videogame Sales

This dataset was chosen as it contains sales data on 16,500 different games over multiple decades and platforms. The dataset was downloaded in CSV format and loaded programmatically with Pandas. The variable below were selected to identify the PC game sales for comparison against engagement in the second dataset.

Type: CSV file

Method: This data was downloaded manually from kaggle.com.

  • Local Directory: Data/Raw/vgsales.csv

Dataset variables:

  • Variable 1: Name: Name of the videogame

  • Variable 2: Genre: Main genre of gameplay, used for analysis of popular games as well as a composite key to identify potential duplicates

  • Variable 3: Publisher: Game publisher, will be used as a composite key to identify potential duplicates

  • Variable 4: Global_Sales: Number of global sales, in millions which also indicates overall order of ranking for the game

Dataset 2: SteamSpy Player Engagement Metrics

This dataset was chosen to retrieve up-to-date information available on modern PC gamers from the SteamSpy API. It allows the perfect complement to the sales dataset as it has data from Steam, the most active, online gaming platform today (e.g. data on owners, playtime, regions, etc.).

Type: Electronic JSON data

Method: "API" method gathered from steamspy.com

  • SteamSpy API: Returns JSON data, no API key required
  • Endpoints:
    1. request=all to retrieve the overall directory and required appid information
    2. appdetails&appid= to retrieve stats for games to match the datasets

Dataset variables:

  • Variable 1: appid: Steam ID to grab game details

  • Variable 2: name: Name of the game on steam

  • Variable 3: publisher: Game publisher, will be used as a join check against dataset 1

  • Variable 4: owners: Number of owners of the game, in a range string

  • Variable 5: average_forever: Overall playtime, in minutes, to determine overall engagement

Derived fields:

  • Variable 6: name_key: A created key field for joining games together based on normalized titles

  • Variable 7/8/9: owners_low; owners_midpoint; owners_high: Numeric midpoint determined from the owners range string

  • Variable 8: hours_engagement = A created field consisting of average_forever / 60

Limitations:

  • Limitation 1: Games identified are for PC/Steam only
  • Limitation 2: SteamSpy provides values (e.g. 'owners') in range estimates and not exact number counts

# Function to update owners to low, mid, and high points
def owners_to_num(s: str):
    new_s = s.replace(",", "" )
    
    num1, num2 = [q.strip() for q in new_s.split("..")]
                  
    num1, num2 = int(num1), int(num2)
    
    midpoint = (num1 + num2) / 2
    
    return num1, midpoint, num2

Answer to research question:
Using a linear Pearson method, this scatter plot shows us there is a weak association (r = approximately .14) between steam engagement playtime and the total, global sales of a game. Meaning, just because a game sells well, it does not mean players on Steam will put in more playtime because of the high sales. There are a few outliers, for instance, of games that had extremely high sales (e.g., > 8 million copies) but rather low average gametime compared to games with far fewer sales (eg. < 1 million copies) but averaged almost 500 hours of playtime. In a linear relationship, the data is right-skewed and the vast majority of games sell less than 1 million copies while giving gamers between 0 and 100 hours of average playtime.

Answer to research question:
As a contrast, when using a proportional rank-based correlation of games with more than 40 hours of gameplay separated into sales bands, the proportion of time spent by players rises as sales bands increase. Meaning, games that sell better are more likely to be engaging and people put more time into the gameplay on average. The positive correlation is still only slight and this is a view of ranked sales, not a straight line correlation.

Overall, both visualizations and analysis provide the same outcome: there is a positive, but fairly weak, relationship between games with many sales receiving more game playtime from owners than games which don't reach as large of an audience.
